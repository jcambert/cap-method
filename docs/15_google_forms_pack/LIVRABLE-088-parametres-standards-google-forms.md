# LIVRABLE-088
# Paramètres standards Google Forms CAP

**Version :** 1.0

---

## Objectif

Ce document définit les paramètres standards à appliquer à tous les Google Forms CAP.

Il garantit une expérience cohérente pour la participante et facilite l'exploitation des réponses par l'accompagnant.

---

## Paramètres généraux

| Paramètre | Valeur recommandée | Commentaire |
|---|---|---|
| Collecte des adresses e-mail | Selon contexte | À activer seulement si nécessaire |
| Limiter à une réponse | Non | Sauf parcours très cadré |
| Autoriser la modification | Oui | Important pour répondre en plusieurs fois |
| Voir le résumé des réponses | Non | Réservé à l'accompagnant |
| Barre de progression | Oui | Utile pour les modules longs |
| Mélanger les questions | Non | L'ordre est important |
| Quiz | Non | CAP n'est pas un test |

---

## Paramètres de présentation

### Titre

Format :

```text
CAP – Module X : Nom du module
```

### Description courte

Inclure :

- objectif du module ;
- temps estimé ;
- rappel qu'il n'y a pas de bonne ou mauvaise réponse ;
- invitation à répondre avec des exemples.

### Sections

Créer une section par partie du livrable source.

Les sections doivent éviter l'effet tunnel.

---

## Paramètres des questions

| Type de question | Usage CAP |
|---|---|
| Réponse courte | Réponse simple, mot-clé, priorité |
| Paragraphe | Réponse narrative ou exemple |
| Choix multiples | Une seule option |
| Cases à cocher | Plusieurs options possibles |
| Échelle linéaire | Intensité de 1 à 10 |
| Grille | Comparaison de scénarios ou critères |
| Date | Date de révision ou action |

---

## Questions obligatoires

Règle générale :

- questions clés : obligatoires ;
- questions sensibles ou longues : facultatives si nécessaire ;
- questions de synthèse : obligatoires ;
- commentaire libre : facultatif.

Le but est d'obtenir assez d'éléments sans bloquer la participante.

---

## Échelles de 1 à 10

Libellé recommandé :

- 1 = très faible / pas du tout
- 10 = très fort / totalement

Ne jamais interpréter un score seul.

Toujours croiser avec les exemples.

---

## Choix « Autre »

Ajouter l'option **Autre** lorsque les choix peuvent ne pas couvrir la réalité de la participante.

Particulièrement recommandé pour :

- valeurs ;
- besoins ;
- motivations ;
- contraintes ;
- ressources ;
- types de soutien ;
- scénarios.

---

## Message de confirmation standard

```text
Merci pour tes réponses.

Je vais les lire tranquillement et revenir vers toi avec une restitution.

Il n'y a pas de bonne ou mauvaise réponse : l'objectif est de mieux comprendre ce qui est juste pour toi.
```

---

## Paramètres de confidentialité

Avant envoi, vérifier :

- le lien n'est pas public au-delà de la participante ;
- les réponses ne sont pas visibles par d'autres ;
- le fichier Google Sheets associé est privé ;
- les données personnelles inutiles ne sont pas demandées ;
- la participante sait pourquoi elle répond.

---

## Paramètres Google Sheets associés

Pour chaque formulaire :

- créer une feuille de réponses dédiée ;
- nommer le fichier clairement ;
- vérifier que la date de réponse est bien collectée ;
- ne pas modifier manuellement les colonnes de réponses brutes ;
- faire les analyses dans des feuilles séparées.

---

## Message d'introduction type

```text
Ce formulaire fait partie du parcours CAP.

Il a pour objectif de t'aider à clarifier certains éléments de ton parcours et de ton projet.

Réponds simplement, avec tes mots.

Tu peux donner des exemples concrets lorsque tu en as.

Il n'y a pas de bonne ou mauvaise réponse.
```

---

## Message si le formulaire est long

```text
Ce module peut prendre un peu de temps.

Tu peux répondre en plusieurs fois si besoin.

L'important n'est pas de faire parfait, mais de répondre de manière sincère et utile pour toi.
```

---

## Contrôle avant duplication

Avant de dupliquer un formulaire modèle pour une participante :

- [ ] Le formulaire modèle est testé.
- [ ] Les sections sont correctes.
- [ ] Les questions sont dans le bon ordre.
- [ ] Les types de questions sont corrects.
- [ ] La modification des réponses est activée.
- [ ] Le message de confirmation est présent.
- [ ] Le formulaire est relié au bon Google Sheets.

---

## Résultat attendu

Tous les formulaires CAP doivent offrir une expérience cohérente, claire et sécurisante.

L'accompagnant doit pouvoir les dupliquer facilement sans reconfigurer toute la structure à chaque accompagnement.
