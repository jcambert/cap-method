# Procédure publication GitHub v1.5.0

## Objectif

Créer manuellement la release GitHub v1.5.0 à partir du dépôt cap-method.

## Étapes

1. Ouvrir le dépôt GitHub : jcambert/cap-method.
2. Aller dans Releases.
3. Cliquer sur Draft a new release.
4. Créer le tag v1.5.0 depuis la branche main.
5. Titre de la release : CAP Method v1.5.0.
6. Copier le contenu de RELEASE_NOTES_v1.5.0.md dans la description.
7. Ajouter les assets exportés si nécessaire.
8. Vérifier le rendu.
9. Publier la release.

## Assets recommandés

- cap_exports_utilisables.zip
- cap_pack_pilote_cap_docx_pdf.zip
- classeur-global-cap.xlsx

## Vérifications avant publication

- README à jour.
- CHANGELOG à jour.
- Avertissement méthodologique présent.
- Licence d'utilisation présente.
- Aucun fichier personnel dans les exports.
- Les exports s'ouvrent correctement.

## Après publication

- Vérifier que la release apparaît bien.
- Vérifier les assets téléchargeables.
- Noter la date de publication.
- Préparer le pilote terrain réel.
