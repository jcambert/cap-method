# LIVRABLE-081
# Protocole pilote CAP

**Version :** 1.0

---

## Objectif

Ce protocole permet de tester la méthode CAP avec une première participante dans des conditions réelles.

Le pilote doit vérifier que la méthode est :

- compréhensible ;
- concrète ;
- tenable dans le temps ;
- utile pour la participante ;
- exploitable pour l'accompagnant ;
- améliorable à partir de retours terrain.

---

## Périmètre du pilote

Le pilote porte sur l'ensemble du parcours CAP :

- accueil ;
- modules 1 à 11 ;
- restitutions intermédiaires ;
- synthèse finale ;
- plan d'action ;
- suivi post-CAP éventuel.

Il peut aussi être réalisé en version courte si la disponibilité de la participante est limitée.

---

## Objectifs du test

Le pilote doit répondre aux questions suivantes :

1. Les consignes sont-elles claires ?
2. Les formulaires sont-ils trop longs ?
3. Les questions produisent-elles des réponses utiles ?
4. Les cahiers de réflexion sont-ils utiles ou trop lourds ?
5. Les restitutions aident-elles la participante à se reconnaître ?
6. Le plan d'action est-il concret ?
7. Le rythme est-il soutenable ?
8. Que faut-il améliorer pour une version 1.2 ?

---

## Sélection de la participante pilote

Critères conseillés :

- personne volontaire ;
- disponible sur plusieurs semaines ;
- capable de répondre par écrit ;
- informée que la méthode est en test ;
- à l'aise avec Messenger et Google Forms ;
- prête à faire un retour honnête.

Éviter pour un premier pilote :

- situation émotionnelle très instable ;
- urgence professionnelle forte ;
- attente de réponse immédiate ;
- confusion entre CAP et thérapie ;
- demande de diagnostic.

---

## Cadre à expliquer

Avant de démarrer, rappeler :

- CAP est une méthode d'accompagnement, pas un test ;
- la participante reste libre de ses réponses ;
- les documents peuvent être ajustés ;
- les retours serviront à améliorer la méthode ;
- elle peut demander à ralentir ;
- elle peut signaler une question mal comprise ;
- elle peut refuser une formulation dans une restitution.

## Version courte possible

Si le pilote complet est trop long, tester en priorité :

1. Module 0 – Accueil
2. Module 1 – Mon histoire
3. Module 2 – Mes valeurs
4. Module 4 – Mes besoins
5. Module 6 – Mes compétences
6. Module 8 – Mon environnement idéal
7. Module 10 – Scénarios
8. Module 11 – Plan d'action
9. Synthèse finale simplifiée

Cette version courte doit être indiquée comme pilote partiel.

---

## Données à collecter pendant le pilote

Pour chaque module, noter :

- temps réel de réponse ;
- questions mal comprises ;
- questions trop longues ;
- questions utiles ;
- émotions ou blocages observés ;
- qualité des réponses obtenues ;
- facilité de restitution ;
- retours spontanés de la participante.

## Critères de réussite du pilote

Le pilote est réussi si :

- la participante comprend le parcours ;
- elle termine au moins la version courte ;
- elle se reconnaît dans plusieurs restitutions ;
- elle repart avec une piste concrète ;
- elle peut nommer une première action ;
- l'accompagnant identifie des améliorations précises.

---

## Critères d'alerte

Ajuster ou ralentir si :

- la participante se sent submergée ;
- les formulaires sont systématiquement trop longs ;
- les restitutions sont mal comprises ;
- le parcours devient trop abstrait ;
- l'action finale reste floue ;
- la participante attend une décision à sa place.

---

## Sortie attendue

À la fin du pilote, produire :

- une fiche retour participante ;
- une fiche retour accompagnant ;
- une grille d'amélioration ;
- un rapport pilote ;
- une liste de modifications pour CAP v1.2.

---

## Décision finale du pilote

À la fin du pilote, choisir :

- [ ] CAP v1.1 validé sans modification majeure
- [ ] CAP v1.2 nécessaire avec ajustements légers
- [ ] CAP v1.2 nécessaire avec refonte partielle
- [ ] Nouveau pilote nécessaire avant diffusion
