# CAP Method® - Google Forms Automation

This folder contains the Google Apps Script generator used to create the CAP Method® Google Forms suite.

## Purpose

The generator creates the operational questionnaire set used during the CAP Method® skills assessment journey.

It creates:

- 10 Google Forms;
- 10 linked Google Sheets response files;
- one Google Drive folder containing all generated resources;
- public form URLs, edit URLs, and response spreadsheet URLs in the Apps Script logs.

## Installation

1. Open Google Apps Script: https://script.google.com/
2. Create a new project.
3. Rename the project, for example: `CAP Method - Google Forms Generator`.
4. Replace the default `Code.gs` content with `cap_method_google_forms_generator.gs`.
5. Save the project.
6. Run the function `createCapMethodFormsSuite`.
7. Accept the requested Google permissions.
8. Open the Apps Script execution logs.

The logs contain the generated URLs for each form and spreadsheet.

## Configuration

The main configuration is located at the top of the script:

```javascript
const CAP_METHOD_CONFIG = {
  brandName: 'CAP Method®',
  folderName: 'CAP Method - Google Forms Suite',
  collectEmail: true,
  requireLogin: false,
  allowResponseEdits: true,
  showProgressBar: true,
  confirmationMessage: 'Merci pour vos réponses. Votre consultant CAP Method® pourra les utiliser pour préparer et personnaliser votre accompagnement.'
};
```

### Recommended production settings

For a professional deployment, review these settings before running the generator:

- `collectEmail`: keep `true` when tracking beneficiaries individually.
- `requireLogin`: set to `true` only if every beneficiary has access to an authorized Google account.
- `allowResponseEdits`: keep `true` during the first operational phase; set to `false` when responses must be frozen after submission.
- `folderName`: change it if multiple environments are needed, for example `CAP Method - Google Forms Suite - Production`.

## Operational workflow

Recommended flow:

1. Generate the suite in a dedicated Google account or shared drive.
2. Review every generated form manually.
3. Configure visual identity in Google Forms if needed.
4. Share beneficiary-facing links only after review.
5. Keep edit links restricted to consultants or administrators.
6. Use the linked Google Sheets as the operational response base.

## Current status

This is the first committed automation version. It is intentionally kept as a single Google Apps Script file because this version has already been tested successfully from the chat-generated prototype.

Future versions should move toward a source-of-truth model where questionnaires are described in JSON or YAML and generated by a reusable builder.

## Roadmap

Planned improvements:
